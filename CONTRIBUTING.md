# Contributing to Connic Composer SDK

Thank you for your interest in contributing to the Connic Composer SDK!

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install development dependencies:

```bash
pip install -e ".[dev]"
```

## Development

### Code Style

We use [Ruff](https://docs.astral.sh/ruff/) for linting and formatting.

```bash
ruff check src/        # Lint
ruff check src/ --fix  # Lint and auto-fix
ruff format src/       # Format
```

## Pull Requests

1. Create a feature branch from `main`
2. Make your changes
3. Add tests for new functionality
4. Ensure all tests pass
5. Submit a pull request

## Reporting Issues

- Use the [GitHub Issues](https://github.com/connic-org/connic-composer-sdk/issues) page
- Include a minimal reproducible example when possible
- Describe expected vs actual behavior

## Code of Conduct

Be respectful and constructive. We're all here to build something great together.
